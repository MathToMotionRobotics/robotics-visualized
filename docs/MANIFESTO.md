# From Math to Motion Manifesto

## Our Mission

From Math to Motion explains how mathematical ideas become physical robot motion through animation, simulation, executable code, and real hardware.

Robotics is often taught as a collection of disconnected subjects:

- mathematics
- mechanics
- electronics
- control
- embedded systems
- software

We connect them into one visible system.

## Our Principles

### 1. Start with a real problem

Every lesson begins with something a robot needs to do, or something that has gone wrong.

We do not begin with an isolated chapter of mathematics.

### 2. Build intuition before abstraction

Animations and physical examples should establish an intuitive model before formal equations are introduced.

### 3. Every equation must produce a visible result

A formula should change a trajectory, move a mechanism, alter a graph, or explain a physical behavior.

Mathematics without an observable consequence does not belong in the main explanation.

### 4. Connect the entire robotics stack

We show how information moves through the complete system:

```text
Goal
→ Planning
→ Kinematics
→ Dynamics
→ Control
→ Motor
→ Mechanism
→ Sensor
→ State Estimation
→ Feedback
```

### 5. Code must be executable

Code shown in an episode should be available, documented, and reproducible.

We do not use decorative code that cannot run.

### 6. Simulation is not reality

We explicitly explain friction, backlash, noise, latency, saturation, calibration errors, structural flexibility, and other differences between mathematical models and physical robots.

### 7. Failure is part of the lesson

Oscillation, incorrect coordinate frames, unstable controllers, communication errors, and mechanical failures are not removed from the story.

They are evidence that helps explain the system.

### 8. One technical standard, multiple languages

The technical content, symbols, code, experiments, and visual language remain consistent across languages.

Translation must preserve meaning rather than copy sentences mechanically.

## What We Are Not

This project is not:

- a collection of disconnected tutorials
- a replacement for rigorous textbooks
- a showcase of equations without experiments
- a maker channel without technical explanation
- a promise to make robotics effortless

Robotics is difficult.

Our goal is not to remove the difficulty. Our goal is to make the difficulty understandable.
